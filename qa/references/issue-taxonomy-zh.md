# QA 问题分类

## 严重程度级别

| 严重程度 | 定义 | 示例 |
|----------|------------|----------|
| **critical** | 阻塞核心工作流、导致数据丢失或使应用崩溃 | 表单提交导致错误页、结账流程中断、数据未经确认被删除 |
| **high** | 主要功能损坏或无法使用，没有替代方案 | 搜索返回错误结果、文件上传静默失败、auth 重定向循环 |
| **medium** | 功能正常但存在明显问题，有替代方案 | 页面加载缓慢（>5s）、缺少表单验证但提交仍有效、仅在移动端布局损坏 |
| **low** | 轻微的装饰或打磨问题 | 页脚拼写错误、1px 对齐问题、hover 状态不一致 |

## 类别

### 1. 视觉/UI
- 布局损坏（元素重叠、文本裁剪、水平滚动条）
- 图片损坏或缺失
- z-index 不正确（元素出现在其他元素后面）
- 字体/颜色不一致
- 动画故障（卡顿、过渡不完整）
- 对齐问题（脱离网格、间距不均匀）
- 深色模式/主题问题

### 2. 功能
- 链接损坏（404、错误的目标）
- 按钮无响应（点击无效果）
- 表单验证（缺失、错误、可绕过）
- 不正确的重定向
- 状态未持久化（刷新或后退按钮时数据丢失）
- 竞态条件（重复提交、过期数据）
- 搜索返回错误或无结果

### 3. UX
- 导航混乱（没有面包屑、死胡同）
- 缺少加载指示器（用户不知道正在发生什么）
- 交互缓慢（>500ms 且无反馈）
- 不清楚的错误消息（"Something went wrong"，没有详细信息）
- 破坏性操作前没有确认
- 跨页面交互模式不一致
- 死胡同（无法返回、没有下一步操作）

### 4. 内容
- 拼写和语法错误
- 过时或不正确的文本
- 遗留的占位符/lorem ipsum 文本
- 截断的文本（没有省略号或"更多"就被截断）
- 按钮或表单字段的标签错误
- 缺失或无帮助的空状态

### 5. 性能
- 页面加载缓慢（>3 秒）
- 卡顿滚动（掉帧）
- 布局偏移（加载后内容跳动）
- 过多网络请求（单个页面超过 50 个）
- 大型未优化图片
- 阻塞的 JavaScript（加载期间页面无响应）

### 6. Console/错误
- JavaScript 异常（未捕获的错误）
- 网络请求失败（4xx、5xx）
- 弃用警告（即将到来的中断）
- CORS 错误
- 混合内容警告（HTTPS 上的 HTTP 资源）
- CSP 违规

### 7. 无障碍
- 图片缺少 alt 文本
- 未标记的表单输入
- 键盘导航损坏（无法 tab 到元素）
- 焦点陷阱（无法逃离模态框或下拉菜单）
- 缺失或不正确的 ARIA 属性
- 颜色对比度不足
- 屏幕阅读器无法访问内容

## 逐页探索清单

在 QA 会话中访问每个页面时：

1. **视觉扫描** — 截取带注释的截图（`snapshot -i -a -o`）。查找布局问题、损坏的图片、对齐问题。
2. **交互元素** — 点击每个按钮、链接和控件。每个的操作是否与描述一致？
3. **表单** — 填写并提交。测试空提交、无效数据、边界情况（长文本、特殊字符）。
4. **导航** — 检查所有进出路径。面包屑、后退按钮、深层链接、移动菜单。
5. **状态** — 检查空状态、加载状态、错误状态、满/溢出状态。
6. **Console** — 交互后运行 `console --errors`。是否有新的 JS 错误或失败的请求？
7. **响应式设计** — 如有需要，检查移动和平板电脑视口。
8. **认证边界** — 注销时会发生什么？不同用户角色呢？
